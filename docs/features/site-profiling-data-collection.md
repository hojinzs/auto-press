# 기능 명세서: 사이트 분석 및 데이터 수집

## 1. 개요
연동된 워드프레스 사이트의 기존 콘텐츠를 수집하고 분석하여, 해당 사이트의 브랜드 보이스(말투), 주요 주제, 타겟 독자층을 파악하는 기능입니다. 분석된 데이터는 향후 AI가 새로운 글을 작성할 때 가이드라인으로 활용됩니다.

## 2. 주요 기능

### 2.1 기존 게시글 데이터 수집
* **설명:** 워드프레스 REST API(`GET /wp/v2/posts`)를 사용하여 이미 발행된 게시글들을 안전하게 가져옵니다.
* **수집 항목:**
  * 게시글 제목 및 본문 (HTML/텍스트)
  * 카테고리 및 태그 정보
  * 발행 일자 및 작성자 정보
* **주요 동작:**
  * 대량의 게시글이 있을 경우 페이지네이션 활용 및 속도 제한(Rate Limiting)을 준수하여 시스템 부하를 최소화합니다.
  * 수집 진행 상태(예: "게시글 50개 중 10개 수집 완료...")를 실시간으로 확인할 수 있습니다.

### 2.2 콘텐츠 임베딩 및 벡터 데이터베이스 저장
* **설명:** 수집된 텍스트 데이터를 LLM(대규모 언어 모델)을 통해 '벡터(수치화된 데이터)'로 변환합니다.
* **주요 동작:**
  * 변환된 벡터 데이터는 Supabase의 `pgvector` 확장을 활용하여 PostgreSQL에 저장됩니다.
  * 이를 통해 단순 키워드 검색이 아닌, '의미 중심'의 고도화된 검색과 분석이 가능해집니다.

### 2.3 사이트 프로파일링 (AI 분석)
* **설명:** 수집된 전체 데이터를 바탕으로 사이트의 정체성을 정의합니다.
* **분석 항목:**
  * **브랜드 보이스:** 친근한지, 전문적인지, 간결한지 등 말투 분석
  * **주요 주제:** 블로그가 다루는 핵심 주제 카테고리화
  * **독자 페르소나:** 어떤 독자들이 좋아할 만한 글인지 추론
* **활용:** 분석 결과는 '사이트 프로필'로 저장되어, 이후 AI 작업 시마다 참고 컨텍스트로 입력됩니다.

### 2.4 시맨틱 중복 체크 및 유사도 검색
* **설명:** 외부 소스에서 새로운 정보를 가져올 때, 내 블로그에 이미 비슷한 내용의 글이 있는지 검사합니다.
* **주요 동작:**
  * Supabase의 벡터 유사도 검색(Vector similarity search) 기능을 사용하여 문맥적으로 중복된 글이 있는지 확인합니다.
  * 중복도가 높을 경우 사용자에게 알림을 주어 유사한 글이 반복 발행되는 것을 방지합니다.

## 3. 시스템 구성 및 기술 (Backend Details)

* **배치 처리 (Batch Processing):** 대규모 데이터 수집 및 임베딩 작업은 서비스 안정성을 위해 비동기 방식으로 처리됩니다.
* **Supabase pgvector 활용:** 고가의 외부 벡터 DB 없이 Supabase 내부에서 효율적으로 벡터 검색을 수행합니다.
* **데이터 접근 격리:** RLS(Row Level Security) 정책을 통해 수집 및 분석된 모든 콘텐츠 데이터는 해당 사용자에게만 접근이 허용됩니다.
