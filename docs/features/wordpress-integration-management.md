# 기능 명세서: 워드프레스 사이트 연동 관리

## 1. 개요
사용자가 자신의 워드프레스 사이트를 본 플랫폼에 원격으로 안전하게 연결하고, 등록된 다수의 사이트를 한곳에서 관리할 수 있도록 지원하는 핵심 기능입니다.

## 2. 주요 기능

### 2.1 사이트 신규 연결 (Add Site)
* **설명:** 사용자의 워드프레스 사이트 주소와 접근 권한 정보를 입력받아 플랫폼과 연동합니다.
* **입력 항목:**
  * 사이트 이름 (사용자가 식별하기 쉬운 별칭)
  * 워드프레스 URL (예: https://myblog.com)
  * 워드프레스 관리자 계정 아이디 (또는 이메일)
  * 워드프레스 애플리케이션 비밀번호 (Application Password)
* **주요 동작:**
  * 정보 입력 후 [연결 테스트] 버튼을 통해 워드프레스 REST API가 정상적으로 응답하는지 확인합니다.
  * 연결 성공 시 사용자의 화면에 성공 메시지를 띄우고, 데이터베이스에 사이트 정보를 안전하게 저장합니다.
  * 연결 실패 시 실패 원인(예: 비밀번호 오타, 주소 오류, 사이트의 REST API 차단 등)을 알기 쉬운 문구로 안내합니다.

### 2.2 사이트 목록 및 상태 관리 (Manage Sites)
* **설명:** 사용자가 등록해 둔 여러 개의 워드프레스 사이트 목록을 확인하고, 정보를 수정하거나 삭제할 수 있습니다.
* **기능 상세:**
  * **목록 보기:** 사이트 이름, URL, 그리고 현재 연동 상태(🟢 정상, 🔴 오류 표시 등)를 리스트 형태로 한눈에 보여줍니다.
  * **수정 기능:** 사이트의 별칭이나 비밀번호가 변경되었을 때 접속 정보를 수정할 수 있습니다.
  * **삭제 (연결 해제):** 더 이상 사용하지 않는 사이트를 목록에서 삭제합니다. 실수로 삭제하는 것을 방지하기 위해 확인 모달(팝업) 창을 띄웁니다.

## 3. 보안 및 정책 (Security & Policy)

* **앱 비밀번호 사용 권장:** 사용자의 워드프레스 메인 로그인 비밀번호 대신, 언제든 폐기할 수 있는 '애플리케이션 비밀번호' 발급 및 사용 방법을 UI 상에서 친절하게 가이드합니다.
* **강력한 암호화 기반 저장:** 입력된 자격 증명(비밀번호 등)은 Supabase 데이터베이스에 철저하게 암호화되어 보관되므로, 관리자조차 원본 비밀번호를 알 수 없습니다.
* **데이터 접근 격리 (RLS):** Supabase의 접근 제어 정책(Row Level Security)을 통해, 다른 가입자가 내 워드프레스 연결 정보를 조회하거나 조작할 수 없도록 완벽히 분리합니다.
* **보안 통신 (HTTPS) 강제:** 데이터 탈취를 막기 위해 `https://`로 시작하는 안전한 접속 주소의 워드프레스 사이트만 연동을 허용합니다.
