# 제품 요구사항 정의서 (PRD) - Supabase 기반 아키텍처 적용 버전

**프로젝트명:** 인텔리전트 워드프레스 콘텐츠 자동화 및 에이전틱 발행 시스템
**문서 버전:** v1.1 (Supabase 통합 고도화)

## 1. 제품 개요 (Product Overview)
본 제품은 '에이전틱 AI(Agentic AI)'를 활용해 사용자의 워드프레스 사이트를 분석(프로파일링)하고, 브랜드 보이스에 맞춘 고품질 SEO 콘텐츠와 멀티미디어를 생성하여 원격으로 발행하는 지능형 SaaS 플랫폼입니다. 특히, 백엔드 및 데이터베이스 인프라로 Supabase를 전면 도입하여 AI 기능(벡터 검색 등)과 실시간 데이터 처리의 효율성을 극대화한 것이 특징입니다.

## 2. 타겟 고객 (Target Audience)
* **콘텐츠 마케터/블로거:** 일관된 톤앤매너로 대량의 콘텐츠를 정기적으로 발행하고자 하는 사용자
* **SEO 전문가:** 자동화된 내부 링크 구축 및 온페이지 SEO 최적화가 필요한 사용자
* **에이전시 및 관리자:** 다수의 워드프레스 사이트를 안전하게 중앙에서 관리하고 모니터링해야 하는 관리자

## 3. 핵심 유저 스토리 (Core User Story)
1. 사용자가 SaaS 플랫폼에 가입한 뒤, 자신이 운영하는 워드프레스를 서비스에 안전하게 연결한다.
2. 서비스가 연결된 워드프레스의 기존 글을 수집하고, 벡터 데이터베이스에 임베딩하여 문체, 주제, 독자층 등을 프로파일링한다.
3. 프로파일링된 가이드라인과 기존 문서 벡터를 바탕으로, AI가 게시글 초안과 시맨틱 내부 링크, 관련 멀티미디어를 생성한다.
4. 사용자가 초안 검토 후 워드프레스로 즉시 발행하거나 예약 발행한다.

## 4. 상세 기능 요구사항 (Functional Requirements)

### 4.1. 사용자 인증 및 워드프레스 안전 연동
* **Supabase Auth 기반 사용자 관리:** 플랫폼 접속 및 계정 관리는 Supabase의 내장 인증(Auth) 시스템을 사용하여 안전하고 빠르게 구축합니다.
* **워드프레스 애플리케이션 비밀번호:** 사용자의 메인 로그인 비밀번호 대신, 워드프레스 '애플리케이션 비밀번호(Application Passwords)'를 이용해 안전하게 REST API 권한만 위임받습니다.
* **HTTPS 강제 및 자격 증명 암호화:** SSL/TLS 암호화 연결만 허용하며, 수집된 API 접근 자격 증명은 Supabase 데이터베이스 내에 암호화되어 안전하게 저장됩니다.

### 4.2. 사이트 프로파일링 및 Supabase를 활용한 데이터 수집
* **기존 콘텐츠 임베딩 및 저장:** GET `/wp/v2/posts` 엔드포인트를 통해 수집된 기존 게시글 데이터는 멀티 LLM을 거쳐 임베딩(Embedding) 벡터로 변환되며, 이 데이터는 Supabase의 pgvector 확장을 통해 PostgreSQL에 저장 및 관리됩니다.
* **AI 시맨틱 검색 기반 중복 제거:** 외부 소스(뉴스 API 등)에서 새로운 콘텐츠를 수집할 때, Supabase의 벡터 유사도 검색(Vector similarity search) 기능을 활용해 기존에 발행된 글과 문맥적 중복(Near-duplicate)이 있는지 사전 필터링합니다.

### 4.3. 지능형 콘텐츠 생성 및 시맨틱 내부 링크 자동화
* **에이전틱 텍스트 생성:** 작업 복잡도에 따라 최적의 LLM을 라우팅하여 초안을 생성합니다. E-E-A-T 기준을 충족하기 위해 표, 리스트 등을 적극 활용합니다.
* **Supabase 기반 시맨틱 내부 링크(Internal Linking):** 새로 작성된 초안의 맥락을 분석한 뒤, Supabase 데이터베이스에 저장된 기존 글들의 임베딩 벡터와 비교하여 가장 연관성이 높은 과거 글을 찾아냅니다. 이를 통해 문맥에 완벽히 맞는 앵커 텍스트로 내부 링크를 자동 삽입하여 SEO와 체류 시간을 극대화합니다.
* **멀티미디어 엔리치먼트:** 텍스트-이미지 생성 AI를 통해 본문 맥락에 맞는 고품질 특성 이미지(Featured Image)와 대체 텍스트(Alt Text)를 함께 생성합니다.

### 4.4. 휴먼 인 더 루프(Human-in-the-loop) 검수 및 실시간 발행 제어
* **80/20 검수 UI:** AI가 80%의 초안을 작성하면, 사용자는 대량 편집(Bulk Edit) 인터페이스에서 최종 20%의 통찰력을 더해 검수합니다.
* **실시간 UI 업데이트:** 대량의 글 생성 및 발행 상태 변화는 Supabase의 실시간 구독(Real-time subscriptions) 기능을 통해 새로고침 없이 사용자 대시보드에 즉각적으로 반영됩니다.

## 5. 비기능 및 시스템 요구사항 (Non-Functional Requirements)

### 5.1. 시스템 아키텍처 (Supabase 집중 통합)
* **단일 데이터베이스에서 RDBMS와 벡터 DB 통합:** 별도의 RDBMS와 벡터 데이터베이스를 나눌 필요 없이, Supabase(PostgreSQL + pgvector) 하나로 일반 트랜잭션 데이터부터 AI 추천 및 의미론적 검색을 위한 임베딩 데이터까지 한 번에 처리하여 인프라 복잡도를 낮춥니다.
* **서버리스 엣지 함수(Edge Functions):** 사용자 위치와 가까운 곳에서 지연 시간(Latency) 없이 AI 모델 API(OpenAI 등)를 호출하고, 워드프레스에 원격 발행 요청을 보내기 위해 Supabase Edge Functions를 적극 활용합니다.

### 5.2. 모니터링 대시보드 및 UX
* **글랜서블 존(Glanceable Zone):** 대시보드 상단에 가동 시간, 금일 발행 통계, API 오류율 등을 한눈에 파악할 수 있도록 시각화합니다.
* **드릴다운(Drill-down) 분석:** 경고 발생 시 해당 지표를 클릭하여 상세 로그로 진입할 수 있는 점진적 공개 인터페이스를 제공합니다.

### 5.3. 보안 및 규정 준수 (Security & Compliance)
* **Row Level Security (RLS):** Supabase의 강력한 RLS 정책을 적용하여, SaaS 내에서 다수의 사용자가 이용하더라도 서로의 워드프레스 데이터나 연결 정보에 접근할 수 없도록 데이터베이스 레벨에서 접근을 철저히 격리합니다.
* 모든 외부 API 연동 시 TLS 1.3 이상을 사용하고, 중요 자격 증명은 안전하게 암호화 보관합니다.

## 6. 성공 지표 (Success Metrics / KPIs)
* **업무 효율성:** 콘텐츠당 리서치, 작성, 발행에 소요되는 시간 단축률 (수동 대비 80% 이상 감소).
* **내부 링크 정확도:** Supabase pgvector 기반의 내부 링크 추천이 사용자에 의해 수정 없이 채택되는 비율 (목표 90% 이상).
* **발행 성공률 및 실시간성:** 예약된 스케줄에 따른 워드프레스 REST API 호출 성공률 (99.9% 이상) 및 Supabase 실시간 동기화를 통한 대시보드 지연 시간 최소화.
* **SEO 성과:** 생성된 글들의 검색 엔진 크롤링 비율 증가 및 유저 체류 시간(Time on Page) 상승.
